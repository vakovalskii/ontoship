🛡️ destructive-guard v0.2.1

PreToolUse-хук: детектит деструктивные shell-команды и вместо глухого блока поднимает подтверждение y/n — работает даже в bypassPermissions. Разбор по токенам, без ложных срабатываний на perform/transform/terraform. Ниже — полный список правил детекта.

CRIT необратимо / бьёт за пределы рабочей копии
ORD локальное обратимое удаление
SKIP осознанно не ловим
В bypassPermissions спрашивается только CRIT; ORD проходит молча. В остальных режимах — любой детект = ask.

Разворачивание и обходы (рекурсивный разбор, глубина ≤4)

  • Снимаются префиксы: sudo command time env nohup builtin exec then do else { ( !
  • Пропускаются присваивания FOO=bar …
  • \rm → снимается ведущий backslash-эскейп
  • bash/sh/zsh/dash/ash/ksh/fish -c "…" → строка разбирается рекурсивно
  • bash <<< "…" (here-string) → разбирается рекурсивно
  • … | xargs CMD → разбирается команда, которую запускает xargs
  • Сегментация по ; && || | & ( ) ` \n — многострочный heredoc-body ловится как отдельные строки

Не ловим (осознанно)

  • > редиректы · docker compose down без -v · git push --force-with-lease
  • Запуск интерпретатора над файлом: python x.py, bash x.sh, psql -f f.sql
  • Деструктив внутри самих скриптов/файлов — чтобы не флажить любой запуск интерпретатора (это убило бы signal-to-noise)
